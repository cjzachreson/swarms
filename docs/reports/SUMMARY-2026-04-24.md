# Session Summary 2026-04-24

## Overview

This session established the initial `SoundSwarms.jl` package architecture and implemented the first working path from swarm simulation through synthetic audio analysis, feature mapping, and diagnostic visualization.

The core design invariant is:

```text
audio source -> audio features -> control mapping -> model parameters -> simulation step -> visualization
```

This keeps the simulation engine independent from audio input, control mapping, and rendering.

## Architecture Decisions

- Package name: `SoundSwarms.jl`.
- Simulation model: fixed-step 2D Vicsek baseline.
- Boundaries: periodic.
- Particle state: positions as a `2 x n` matrix and headings as a vector.
- Randomness: explicit `AbstractRNG` arguments for deterministic tests.
- Audio interface: normalized `AudioFeatureFrame`s, independent of raw audio format.
- Feature history: finite-length `AudioFeatureBuffer`.
- Control mapping: first fixed mapping is `rms -> speed`, `high_band -> noise`.
- Speed safety: mapped speed is bounded so `speed * dt <= alignment_radius`.
- Visualization: dependency-free browser HTML diagnostics before real-time backend work.
- Generated outputs are ignored under `outputs/`.

## Implemented Modules

### Simulation

Implemented core simulation types and baseline Vicsek dynamics:

- `SwarmState`
- `SwarmParameters`
- `initialize_swarm`
- `step!`

The Vicsek update includes self-neighbor alignment, circular mean headings, uniform angular noise, periodic wrapping, and minimum-image periodic neighbor distances.

### Audio

Implemented the audio feature layer:

- `AudioFeatureFrame`
- `AudioFeatureBuffer`
- `push_feature!`
- `latest_feature`
- `buffer_capacity`
- `synthetic_feature_frame`
- `synthetic_feature_frames`

Then added the first raw synthetic audio signal path:

- `AudioSampleFrame`
- `AudioSpectrumFrame`
- `synthetic_sweep_sample_frames`
- `analyze_sample_frame`
- `analyze_sample_frames`

The current analyzer is dependency-free and uses a small direct DFT for fixture-scale band and centroid estimates. It is not intended as the final real-time audio analysis implementation.

### Control

Implemented the first feature-to-parameter mapping:

- `FeatureParameterMapping`
- `map_features_to_parameters`

Default mapping:

```text
rms -> speed
high_band -> noise_strength
```

The synthetic raw-signal demo uses:

```text
rms -> speed
spectral_centroid -> noise_strength
```

### Visualization

Implemented dependency-free HTML/canvas visualization:

- `SwarmFrame`
- `write_html_animation`
- `write_diagnostic_html_animation`

Diagnostic visualization can show:

- swarm particles with trails
- selected controlling feature traces only
- optional synchronized frequency-spectrum heatmap

### Runtime

Extracted the offline controlled simulation loop:

- `ControlledRunFrame`
- `run_controlled_simulation`

This runner owns the current orchestration path:

```text
initial state + audio frames + mapping -> synchronized swarm/audio/parameter frames
```

It copies the initial state before running, so callers keep their original state unchanged.
