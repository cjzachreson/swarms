## Project Context

We are building `SoundSwarms.jl`, a Julia package for audio-reactive 2D swarm simulation. The target use case is eventually live party/music visualizations driven by microphone or recorded audio, but current development uses deterministic synthetic inputs.

Core invariant:

```text
audio source -> audio features -> control mapping -> model parameters -> simulation step -> visualization
```

The simulation should stay independent from raw audio IO and rendering.

## Current State

The package now has working modules for:

- `Simulation`: state, parameters, initialization, baseline Vicsek update.
- `Audio`: normalized feature frames, feature buffers, synthetic features, raw sample frames, basic DFT analysis.
- `Control`: feature-to-parameter mapping.
- `Visualization`: dependency-free HTML/canvas animation and diagnostic views.
- `Runtime`: reusable offline controlled simulation runner.

The current tested path is:

```text
synthetic sweep samples
-> windowed sample frames
-> DFT-based feature/spectrum analysis
-> feature-to-parameter mapping
-> controlled Vicsek simulation
-> diagnostic HTML with swarm, feature traces, spectrum heatmap
```
